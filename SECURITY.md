# Security Policy

If you find a privacy leak, secret exposure pattern, or a misleading audit rule in this repo, please report it privately before opening a public issue.

## What to report

- exposed secrets or token-like strings
- false negatives in the audit helper for obvious privacy leaks
- instructions that could encourage unsafe open-source publication
- metadata or packaging guidance that creates a security risk

## Preferred process

Open a private security report through GitHub security reporting if available for this repo. If not, contact the maintainer privately first and include:

- affected file or path
- reproduction steps
- expected safe behavior
- observed behavior

Please avoid publishing working secrets or sensitive data in public issues.
